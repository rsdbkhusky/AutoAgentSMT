## 第5页 | 项目背景与目的

- **研究动机**：传统金融仿真依赖预设规则，无法模拟交易者的认知差异与情绪博弈
- **核心问题**：能否用大语言模型赋予每个交易Agent独立的"人格"和"判断力"，让市场涌现出真实的供需博弈？
- **项目目标**：
  - 构建一个Agent驱动的完整股市生态系统
  - 验证LLM Agent在多角色博弈场景中的行为多样性
  - 探索"感知—决策—行动"范式在金融仿真中的应用

## 第6页 | 什么是Agent？

Agent（智能体）是能够自主感知环境、做出决策并执行行动的AI实体。与传统的单次问答不同，Agent具备：

- **自主性**：根据环境变化独立决策，无需人工干预
- **记忆性**：保留历史交易和事件记忆，影响后续判断
- **交互性**：通过群聊相互影响，形成信息传导链
- **个性化**：由配置文件定义性格、策略、信息偏好，行为完全不同

## 第10页 | Agent设计：配置驱动的多样性

系统采用"一个Agent类 + 多份配置文件"的设计，无需为每种性格编写子类。

每个Agent的YAML配置定义：人物背景故事（500-1000字）、初始资金、风险偏好、信息接收频道。同一份配置可实例化多个Agent，也可为单个角色创建独立配置。

当前系统包含4种通用类型（保守/激进/跟风/价值）和18个独立角色人物，共30个Agent。

## 第14页 | 撮合引擎

撮合引擎是市场的核心，完全模拟真实交易所规则：

- **价格优先、时间优先**：出价高的买单优先成交，先到的同价订单优先
- **涨跌停限制**：基于前收盘价±10%
- **订单簿管理**：维护每只股票的买卖挂单队列
- **K线聚合**：每回合结束后根据实际成交聚合OHLCV数据
- **盘口深度**：实时提供买五卖五档位信息

所有价格波动完全由Agent交易行为产生，无任何人工干预。

## 第16页 | 并发调度与性能

30个Agent使用`asyncio.gather`真并发调用LLM API，配合令牌桶限速器控制调用频率。

```
simulation_loop 每回合流程：
  推进虚拟时钟 → 生成新闻 → 获取市场快照
  → 30个Agent并发(感知→决策→行动)
  → 聚合K线 → WebSocket广播更新
  → 等待间隔 → 进入下一回合
```

单回合内所有Agent同时向LLM发起请求，利用异步IO最大化吞吐量，实际受限于API供应商的并发能力。

## 第18页 | 前端功能：社交与资讯

交流群面板实时展示Agent之间的观点碰撞——看多者激情呐喊、看空者冷静分析、逆向者故意唱反调，呈现出真实股民群聊的生态。

资讯中心按时间线滚动显示LLM生成的财经新闻，每条标注情绪倾向（利好/利空/中性）和受影响股票，Agent根据自身信息偏好选择性接收这些新闻。

前端通过WebSocket实时接收推送，辅以5秒轮询保底，确保数据不丢失。
